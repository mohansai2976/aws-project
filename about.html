Behind the build

Built in phases. Verified at every step.

This project was built as a hands-on introduction to production-style AWS deployment patterns — private origins, CDN delivery, federated identity, CI/CD, and active monitoring — without taking on any ongoing cost.

Build log

Seven phases, one checkpoint each

PHASE 01

Storage layer — Amazon S3

Private bucket created with all Block Public Access settings on. Static website hosting deliberately left disabled — it requires a public bucket, which this design avoids entirely.

PHASE 02

Delivery layer — Amazon CloudFront

Distribution created with Origin Access Control, so CloudFront — and only CloudFront — can read from the bucket. Placed on the $0/month Free flat-rate plan.

PHASE 03

Identity layer — IAM OIDC

An OIDC identity provider trusting GitHub was registered, and a role scoped to this exact repository and branch was created with a least-privilege inline policy.

PHASE 04

CI/CD — GitHub Actions

A workflow was added to sync files to S3 and invalidate the CloudFront cache on every push — authenticated with zero stored credentials.

PHASE 05

Monitoring & alerting — CloudWatch and SNS

A CloudWatch alarm on CloudFront's 5xx error rate was wired to an SNS topic, so a real infrastructure fault triggers an email — verified with a controlled manual test rather than a live outage.

PHASE 06

Documentation

Screenshots, an architecture diagram, and a full written report were captured before teardown, so the project remains reviewable indefinitely.

PHASE 07

Teardown

All resources were deleted after review — CloudFront disabled then removed, S3 emptied, IAM role and OIDC provider cleaned up, CloudWatch alarm and SNS topic deleted.

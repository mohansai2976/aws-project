Deployment flow

Push to main. Everything else is automatic.

A single GitHub Actions workflow authenticates via OIDC, syncs files to S3, and invalidates the CloudFront cache — in that order, every time.

01
git push
Code pushed to the main branch on GitHub.
02
GitHub Actions
Assumes an IAM role via OIDC — no stored secrets.
03
Amazon S3
Files synced to a private bucket. Public access blocked.
04
CloudFront
Cache invalidated. Fresh content served over HTTPS.
05
CloudWatch
Watches CloudFront's 5xx error rate continuously.
06
SNS
Emails the team the moment an alarm fires.
Architecture

Five services. One job each.

No custom domain, no purchased certificate — CloudFront's default HTTPS keeps this stack genuinely free.

STORAGE

Amazon S3

Private object storage for HTML, CSS, and JS. Block Public Access stays on — only CloudFront can read it.

DELIVERY

Amazon CloudFront

Global CDN and the site's only public entry point. Free HTTPS via its default domain certificate.

IDENTITY

AWS IAM (OIDC)

Issues short-lived, scoped credentials to GitHub Actions. No access keys stored anywhere, ever.

MONITORING

Amazon CloudWatch

Continuously evaluates CloudFront's error rate against a defined threshold.

ALERTING

Amazon SNS

Delivers a real email the moment CloudWatch detects a genuine error spike.

CI/CD

GitHub Actions

Syncs files to S3 and invalidates the CloudFront cache on every push to main.
